# Status Protocol

Before pausing work:

1. Update `docs/status/current.md`.
2. Update `docs/status/roadmap.md` when the current focus, immediate plan, or
   later planned work changes.
3. Update `docs/status/implementation-matrix.md` when implemented capability or
   planned capability changes.
4. Append one entry to `docs/status/checkpoints.md`.
5. Record the exact verification command and result.
6. Record any uncommitted files intentionally left behind.

Keep entries factual and short.
